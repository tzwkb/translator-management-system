# 译员管理系统 Demo

译员全生命周期 + 结算系统的可运行演示。技术栈与开发文档一致：FastAPI + SQLAlchemy + SQLite + 单文件 HTML/ECharts，与「部门看板」同一套底座思路。

## 跑起来

本机 Python 是 externally-managed（PEP 668），用 venv：

    cd demo
    python3 -m venv .venv                          # 首次
    .venv/bin/pip install -r requirements.txt      # 首次
    .venv/bin/python app.py                         # 每次启动

浏览器开 http://127.0.0.1:8000（这台机器上 `.venv` 已建好、依赖已装，直接跑第三行即可）。

数据存在 `demo.db`（首次启动自动建表 + 写样例）。清空重来删 `demo.db` 和 `.key`。
顶栏右上角「视角」可切 资源端 / 财务 / 资源端Agent / boss。

## 做了什么（全部可用、已测）

- **译员主表**：列表、搜索、新增、编辑、软删除；Excel 导入导出；**邮箱唯一**约束
- **报价变更（留痕）**：调价历史；翻译类调价自动同步主表当前费率
- **质量记分卡**：评分、错误等级、评审人；**从 LQE 报告导入**（读 LQE 汇总 sheet，按子表名匹配译员）
- **合同**：协议 + NDA + 到期；30 天内到期在概览橙色告警
- **支付信息**：银行账号、身份证 **AES-256 加密存储 + 脱敏展示**，「查看明文」需可编辑视角并记审计
- **客诉**：类型、严重度、扣款、处理
- **产能**：按周占用、可用率 = 100 − 占用，超 100% 标红（利用率口径待 PM，未计算）
- **PO 结算**：按月账单、待付/已付分币种汇总、金额验算（字数×单价对不上标红）、状态流转
- **概览**：在册 / 活跃 / 休眠 / 30 天内合同到期
- **角色与权限（后端强制）**：资源端·财务可编辑，boss 只读；只读视角的写操作直接 403
- **Agent 接口**：服务账号 `资源端Agent`；档期等低风险直接落，**费率/PO 进待审队列**，资源端在「待审核」页批准/驳回才生效（钱必须人工核验）
- **审计日志**：所有写操作记录谁/何时/改了什么，区分人与 agent
- **Langlobal 配色 + logo**（base64 内嵌，不受代理影响）

配套自动化 skill 在 `../translator-mgmt-agent/`：从微信/企微抓译员动态 → 写系统（client.py）。

## 测试

后端验收（27 项：RBAC / 边界 / PRD 硬规则 / 安全 / agent 护栏）：

    .venv/bin/python app.py              # 一个终端起服务
    .venv/bin/python test_acceptance.py  # 另一个终端跑

前端用 Playwright 冒烟（全标签页 + 6 个详情子表 + 查看明文 + agent 待审批准 + logo），开发时跑过，无 JS 报错。
